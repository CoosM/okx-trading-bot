## История проекта

Этот проект — результат **11 дней непрерывной разработки**.

На протяжении этих 11 дней я работал **примерно по 18 часов каждый день**, изучая документацию, исправляя ошибки, переписывая архитектуру и постепенно собирая систему из множества отдельных компонентов.

Сейчас она уже более **2 месяцев работает на реальном аккаунте**, и результаты остаются положительными.

---

# Цель проекта

Полностью автоматизировать исполнение собственной DCA-стратегии.

Идея была простой:

* TradingView принимает решение.
* Бот автоматически исполняет сделку.
* Система сама ведёт состояние позиции.
* Всё работает без ручного вмешательства.

Теперь стратегия работает вместо меня.

# Основные возможности

## TradingView

* автоматические BUY сигналы
* автоматические SELL сигналы
* RSI фильтр
* Engulfing фильтр
* тестовые BUY/SELL сигналы
* ручная отправка шага DCA
* выбор биржи

---

## Flask API

* приём webhook
* обработка JSON
* маршрутизация действий
* проверка состояния
* Health endpoint

---

## Bitget

* Market Buy
* Market Sell
* получение баланса
* автоматический расчёт количества
* округление по precision
* ограничение количества DCA шагов

---

## OKX

* Market Buy
* Market Sell
* получение баланса
* автоматический расчёт объёма продажи
* отдельный счётчик шагов

---

## GitHub Gist

Используется как удалённое постоянное хранилище.

Хранится:

* состояние DCA
* количество шагов
* Bitget state
* OKX state

После перезапуска сервера данные не теряются.

---

## Кэширование

Используется внутренний cache.

Если GitHub временно недоступен:

* бот продолжает работать
* используется последнее известное состояние
* после восстановления данные снова синхронизируются

---

## Управление DCA

* ручная установка step
* автоматическое увеличение step после BUY
* автоматическое уменьшение после SELL
* ограничение максимального количества шагов
* независимое управление Bitget и OKX
